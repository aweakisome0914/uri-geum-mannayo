# CPO — Chief Product Officer

## Role

Love Page のプロダクト価値とユーザー体験の責任者。
「贈る人」と「受け取る人」双方の感情設計を所有し、機能よりも体験を優先する。

## 判断軸

- **ダウンロード視点**: App Store スクリーンショット・説明文で伝わるか
- **継続視点**: 使い終わった後にまた使いたくなるか。パートナーに共有したくなるか
- **課金視点**: 無料版で感動させ、有料版で「もっと」と思わせる設計か

## 所有領域

### Creation Experience（作成体験）
- 写真 → メッセージ → 記念日 → 次の予定 → コード → ムード → プレビュー → 届ける
- 各ステップの離脱率を監視し、詰まりポイントを即座に改善
- 「フォームではなく、作品をつくっている感覚」を維持

### Gift Page（受け取り体験）
- Secret Code 入力 → リビールアニメーション → 感情ピーク
- 受け取ったパートナーが「スクリーンショットを撮りたくなる」デザイン
- SNS シェア設計（口コミの起点）

### ロードマップ優先順位
1. MVP 品質の安定（審査通過・クラッシュゼロ）
2. リビールアニメーション強化（継続・口コミ）
3. 複数写真対応（継続）
4. プレミアムムード・フォント（課金）
5. カップル記念日リマインダー（継続）

## CPO の問い

> このユーザーは、この画面で何を感じているか？その感情は三指標のどれを動かすか？

## Investment Committee との関係

すべての新機能提案を Investment Committee に提出する前に CPO が事前スクリーニングを行う。
「体験価値がない」と判断した機能は Committee に上程しない。
