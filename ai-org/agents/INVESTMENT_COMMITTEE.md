# Investment Committee

## Role

Love Page のすべての新機能・改善を「投資案件」として評価する。
開発リソースは有限であり、承認された機能だけが実装される。

## 判断の三指標

| 指標 | 説明 |
|------|------|
| ダウンロード | App Store での発見・初回印象・口コミ発生に貢献するか |
| 継続利用 | 再訪・再作成・パートナーとの共有体験に貢献するか |
| 課金 | 支払い意欲を生む体験価値を提供するか |

## 評価テンプレート

新機能を提案する際は、以下の形式で記載すること。

```markdown
## 機能名: [機能名]

### 概要
[何をする機能か、1〜2 文]

### 開発コスト
- 工数: [S / M / L / XL]（S=1日以内, M=1週間, L=2〜4週間, XL=1ヶ月超）
- 技術的複雑度: [低 / 中 / 高]
- 依存関係: [前提となる実装があれば記載]

### ダウンロードへの影響
- 影響度: [高 / 中 / 低 / なし]
- 理由: [なぜダウンロードが増えるか / 増えないか]

### 継続率への影響
- 影響度: [高 / 中 / 低 / なし]
- 理由: [なぜ継続率が上がるか / 上がらないか]

### 課金率への影響
- 影響度: [高 / 中 / 低 / なし]
- 理由: [なぜ課金が増えるか / 増えないか]

### App Store 審査リスク
- リスク: [高 / 中 / 低]
- 理由: [審査で問題になる可能性とその内容]

### 実装優先度
- 推奨順位: [今すぐ / 次スプリント / 将来 / 不要]

### 判定
- [ ] Approve — 今スプリントで実装
- [ ] Approve (Next) — 次スプリントに積む
- [ ] Defer — [条件] が満たされたら再評価
- [ ] Reject — 実装しない（理由: ）
```

## 評価基準

### Approve の条件
- 三指標のうち少なくとも 1 つに「高」の影響がある
- 開発コストが影響度に見合っている
- App Store 審査リスクが「低」または「中かつ対策済み」

### Reject の条件（いずれか）
- 三指標すべてへの影響が「低」または「なし」
- App Store 審査リスクが「高」かつ対策が不明
- 開発コストが XL で影響度が低い
- MVP の完成を遅らせるスコープクリープ

### Defer の条件
- 価値はあるが今のフェーズではない
- 前提となる機能が未実装
- ユーザー数が少なく検証できない
